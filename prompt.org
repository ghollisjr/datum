**1. `sql-datum--fetch-columns-async` watcher still uses `maphash`**

In `sql-datum.el`, the watcher installed by `sql-datum--fetch-columns-async` fires on every piece of process output and does a full `maphash` scan over `sql-datum--columns` to check whether the table has been populated. This is O(n) per output chunk for each in-flight fetch.

Replace the `maphash` scan in the watcher with a direct string match against the raw output:

```elisp
(letrec ((watcher
          (lambda (output)
            (when (string-match-p
                   (concat "##DATUM:introspect:columns:"
                           (regexp-quote table) ":")
                   output)
              (remove-hook 'comint-output-filter-functions watcher t)
              (remhash table pending-hash)))))
  ...)
```

The preoutput filter runs before `comint-output-filter-functions`, so by the time the watcher fires the envelope has already been processed and the column cache populated. Matching on the raw envelope string in the watcher is both faster and more precise ŌĆö it fires exactly once on the relevant output chunk without any hash scanning.

**3. `quote_identifier` method on `BaseDriver`**

In `datum/importer.py`, `_build_ddl` uses a hardcoded check for postgres vs. everyone-else to decide how to quote column names:

```python
if driver and driver.dialect_name == "postgres":
    cols = f'"{name}"...'
else:
    cols = f"[{name}]..."
```

Square bracket quoting is MSSQL-specific. MySQL requires backticks, Oracle uses double-quotes, SQLite accepts double-quotes or no quoting. Add a `quote_identifier(name)` method to `BaseDriver` and override it in each driver:

- `base.py` / `AnsiDriver` ŌĆö double-quote: `f'"{name}"'`
- `mssql.py` ŌĆö square bracket: `f'[{name}]'`
- `postgres.py` ŌĆö double-quote: `f'"{name}"'`
- `mysql.py` ŌĆö backtick: `f'`{name}`'`
- `oracle.py` ŌĆö double-quote: `f'"{name}"'`
- `sqlite.py` ŌĆö double-quote: `f'"{name}"'`

Update `_build_ddl` to call `driver.quote_identifier(name)` for each column, falling back to double-quote when `driver` is `None`. Also audit any other place in `importer.py` or `commands.py` that hardcodes identifier quoting and update those to use the method.

**4. `sql-datum--prompt-suppressed` race condition comment**

In `sql-datum.el`, `sql-datum--prompt-suppressed` is a single buffer-local boolean used by both the refresh chain and `sql-datum--fetch-columns-async` to detect command completion. If both are in flight simultaneously, the first suppressed prompt sets the flag, and a watcher belonging to the *other* command may see it and consider itself done prematurely.

This is a subtle correctness issue that is difficult to fix fully without Python-side changes (e.g. a correlation token per command). For now, add a clearly written comment at the `defvar-local` declaration of `sql-datum--prompt-suppressed` explaining:

- What the flag is for
- Why it's a single boolean rather than a counter or queue
- The known race condition with concurrent silent commands
- What a full fix would require (correlation tokens, or serializing all silent commands through the refresh chain mechanism)

Also add a comment in `sql-datum--fetch-columns-async` noting that it runs independently of the refresh chain and is therefore susceptible to the race in practice, and that the `sql-datum--at-prompt-p` guard reduces but does not eliminate the window.
