** TODO =datum-send-and-wait=
Send a command string to the datum process and block until completion.

Implementation sketch: enqueue a transaction with a =:done-fn= that
sets a flag, then wait on the flag using =accept-process-output=:

#+begin_src elisp
(defun datum-send-and-wait (cmd &optional timeout)
  "Send CMD to datum and block until completion or TIMEOUT seconds."
  (let ((done nil)
        (deadline (+ (float-time) (or timeout 120)))
        (buf (sql-find-sqli-buffer 'datum)))
    (with-current-buffer buf
      (sql-datum--enqueue-one cmd
        :done-fn (lambda () (setq done t))))
    (let ((proc (get-buffer-process buf)))
      (while (and (not done) (< (float-time) deadline))
        (accept-process-output proc 0.05)))
    (unless done
      (error "datum-send-and-wait: timed out after %ss" (or timeout 120)))))
#+end_src

- On =error= envelope, signals an Emacs error so callers can use
  =condition-case=
- Non-blocking variant =datum-send-async= with callback for cases
  where waiting is unnecessary
- Usable from =org-babel= source blocks for literate ETL scripts

** Example ETL patterns
#+begin_src elisp
;; Move rows between databases
(datum-send-and-wait ":use source_db")
(let ((ids (datum-query-column "SELECT id FROM pending_orders")))
  (datum-send-and-wait ":use target_db")
  (dolist (id ids)
    (datum-send-and-wait
     (format "INSERT INTO archive SELECT * FROM source.orders WHERE id = %s" id))))

;; Conditional ETL with scalar check
(let ((count (datum-query-scalar "SELECT COUNT(*) FROM staging")))
  (when (> count 0)
    (datum-send-and-wait "EXEC dbo.process_staging")
    (message "Processed %d rows" count)))
