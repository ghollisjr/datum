#+TITLE: Datum Test Guide
#+DESCRIPTION: Running the test suite and setting up test databases

* Quick start (Docker)

The easiest way to run all tests is with Docker Compose.  It provisions
MariaDB, PostgreSQL, SQL Server, and SQLite automatically — no manual
database setup required.  Oracle 23c Free can be included via the
=oracle= Docker Compose profile.

#+BEGIN_SRC sh
# Run the full test suite (unit + integration + Emacs)
docker compose up --build --abort-on-container-exit test-runner

# Clean up containers and volumes
docker compose down -v

# Include Oracle (adds ~1.5 GB download on first run):
COMPOSE_PROFILES=oracle DATUM_BACKENDS=mysql,postgres,mssql,sqlite,ansi,oracle \
  docker compose up --build --abort-on-container-exit test-runner
#+END_SRC

That's it.  The =test-runner= container installs datum with all ODBC
drivers, waits for every database to be healthy, and runs =tests/run-all.sh=.
Strict mode (=DATUM_STRICT=1=) is enabled in Docker, so any backend that
fails to connect or is missing a driver causes a hard failure — no silent
skips.

** Docker services

| Service       | Image                                  | Port (host→container) | Purpose                                |
|---------------+----------------------------------------+-----------------------+----------------------------------------|
| =mysql=       | =mariadb:11=                           | 3307→3306             | MariaDB with =test-mysql.sql= loaded   |
| =postgres=    | =postgres:16=                          | 5433→5432             | PostgreSQL with =test-postgres.sql=     |
| =mssql=       | =mcr.microsoft.com/mssql/server:2022=  | 1434→1433             | SQL Server 2022                        |
| =mssql-init=  | (same MSSQL image)                     | —                     | One-shot: loads =test-mssql.sql=        |
| =oracle=      | =gvenzl/oracle-free:23-slim-faststart=  | 1522→1521             | Oracle 23c Free (profile: oracle)      |
| =oracle-init= | (same Oracle image)                    | —                     | One-shot: loads =test-oracle.sql=       |
| =test-runner= | Custom (=docker/test-runner/Dockerfile=)| —                     | Runs the test suite                    |

Host ports are offset by 1 to avoid clashing with local database servers.

** Using Docker databases for local development

Start just the databases (no test runner):

#+BEGIN_SRC sh
docker compose up -d mysql postgres mssql mssql-init

# Include Oracle:
COMPOSE_PROFILES=oracle docker compose up -d mysql postgres mssql mssql-init oracle oracle-init
#+END_SRC

Then run tests or datum locally against them:

#+BEGIN_SRC sh
# Run integration tests against Docker databases
DATUM_BACKENDS=mysql,postgres,mssql,sqlite,ansi \
DATUM_MYSQL_SERVER=localhost DATUM_MYSQL_USER=root DATUM_MYSQL_PASS=datum_test \
DATUM_PG_SERVER=localhost DATUM_PG_PORT=5433 DATUM_PG_USER=postgres DATUM_PG_PASS=datum_test \
DATUM_MSSQL_SERVER=localhost DATUM_MSSQL_PORT=1434 DATUM_MSSQL_USER=sa DATUM_MSSQL_PASS='DatumTest1!' \
  bash tests/python/test_integration.sh

# Include Oracle (requires Oracle Instant Client + ODBC driver installed locally):
DATUM_BACKENDS=mysql,postgres,mssql,sqlite,ansi,oracle \
DATUM_MYSQL_SERVER=localhost DATUM_MYSQL_USER=root DATUM_MYSQL_PASS=datum_test \
DATUM_PG_SERVER=localhost DATUM_PG_PORT=5433 DATUM_PG_USER=postgres DATUM_PG_PASS=datum_test \
DATUM_MSSQL_SERVER=localhost DATUM_MSSQL_PORT=1434 DATUM_MSSQL_USER=sa DATUM_MSSQL_PASS='DatumTest1!' \
DATUM_ORACLE_SERVER=localhost DATUM_ORACLE_PORT=1522 DATUM_ORACLE_USER=datum_test DATUM_ORACLE_PASS=datum_test \
  bash tests/python/test_integration.sh
#+END_SRC

Or connect interactively:

#+BEGIN_SRC sh
# MariaDB
datum --sql-type mysql \
  "Driver={MariaDB};Server=localhost;Port=3307;Database=datum_test;User=root;Password=datum_test"

# PostgreSQL
datum --sql-type postgres \
  "Driver={PostgreSQL};Server=localhost;Port=5433;Database=datum_test;Uid=postgres;Pwd=datum_test"

# SQL Server
datum --sql-type mssql \
  "Driver={ODBC Driver 18 for SQL Server};Server=localhost,1434;Database=datum_test;Uid=sa;Pwd=DatumTest1!;TrustServerCertificate=yes"

# Oracle (requires COMPOSE_PROFILES=oracle)
datum --sql-type oracle \
  "Driver={Oracle 23 ODBC driver};DBQ=localhost:1522/FREEPDB1;Uid=datum_test;Pwd=datum_test"
#+END_SRC

* Manual database setup (without Docker)

If you prefer to run databases natively, load the test SQL files manually.

** PostgreSQL

#+BEGIN_SRC sh
psql -U postgres -f test-postgres.sql
#+END_SRC

Connect with datum:
#+BEGIN_SRC sh
datum --sql-type postgres \
  "Driver={PostgreSQL};Server=localhost;Port=5432;Database=datum_test;Uid=postgres;Pwd=YOUR_PASSWORD"
#+END_SRC

** MySQL / MariaDB

#+BEGIN_SRC sh
sudo mysql < test-mysql.sql
#+END_SRC

Or with password prompt:
#+BEGIN_SRC sh
mysql -u root -p < test-mysql.sql
#+END_SRC

Connect with datum:
#+BEGIN_SRC sh
datum --sql-type mysql \
  "Driver={MariaDB};Server=localhost;Port=3306;Database=datum_test;Uid=root;Pwd=YOUR_PASSWORD"
#+END_SRC

** SQL Server

#+BEGIN_SRC sh
sqlcmd -S localhost -U sa -P 'YOUR_PASSWORD' -C -i test-mssql.sql
#+END_SRC

Connect with datum:
#+BEGIN_SRC sh
datum --sql-type mssql \
  "Driver={ODBC Driver 18 for SQL Server};Server=localhost;Database=datum_test;Uid=sa;Pwd=YOUR_PASSWORD;TrustServerCertificate=yes"
#+END_SRC

** Oracle

Requires Oracle 23c with an app user =datum_test=.  The easiest way is
via Docker:

#+BEGIN_SRC sh
docker run -d --name oracle-test \
  -e ORACLE_PASSWORD=datum_test \
  -e APP_USER=datum_test -e APP_USER_PASSWORD=datum_test \
  -p 1522:1521 gvenzl/oracle-free:23-slim-faststart

# Wait for the container to be healthy (~30-60s on first start):
until docker exec oracle-test healthcheck.sh &>/dev/null; do sleep 5; done

# Load test data:
docker exec -i oracle-test sqlplus datum_test/datum_test@localhost/FREEPDB1 < test-oracle.sql
#+END_SRC

Connect with datum:
#+BEGIN_SRC sh
datum --sql-type oracle \
  "Driver={Oracle 23 ODBC driver};DBQ=localhost:1522/FREEPDB1;Uid=datum_test;Pwd=datum_test"
#+END_SRC

** SQLite

#+BEGIN_SRC sh
bash test-sqlite.sh
#+END_SRC

Connect with datum:
#+BEGIN_SRC sh
datum --sql-type sqlite "Driver={SQLite3};Database=/tmp/datum_test.db"
#+END_SRC

* Test suite structure

The test suite has three layers, run by =tests/run-all.sh=:

| Layer             | Runner                           | What it tests                           |
|-------------------+----------------------------------+-----------------------------------------|
| Python unit tests | =python -m pytest tests/python/= | Driver registry, SQL type detection     |
| Integration tests | =tests/python/test_integration.sh= | Live database introspection & queries |
| Emacs tests       | =tests/emacs/run-tests.sh=       | Envelope protocol, Emacs-side parsing   |

** Integration test backends

The integration test script auto-detects available ODBC drivers and skips
backends that aren't reachable.  Control which backends to test:

#+BEGIN_SRC sh
# Test only PostgreSQL
DATUM_BACKENDS=postgres bash tests/python/test_integration.sh

# Test the five default backends
DATUM_BACKENDS=mysql,postgres,mssql,sqlite,ansi bash tests/python/test_integration.sh

# Test all six (including Oracle)
DATUM_BACKENDS=mysql,postgres,mssql,sqlite,ansi,oracle bash tests/python/test_integration.sh
#+END_SRC

Default order: =mysql=, =postgres=, =mssql=, =sqlite=, =ansi=.
Oracle is not in the default set — add it explicitly via =DATUM_BACKENDS=.

The =ansi= backend reuses the PostgreSQL connection but forces
=--sql-type=ansi=, verifying that the generic INFORMATION_SCHEMA fallback
driver works against a real database.

** Strict mode

Set =DATUM_STRICT=1= to make any skip a fatal error.  This is enabled by
default in the Docker test runner to ensure all backends are exercised.
Locally, strict mode is off — unavailable backends are skipped gracefully.

#+BEGIN_SRC sh
# Fail if any backend can't connect or is missing a driver
DATUM_STRICT=1 bash tests/python/test_integration.sh
#+END_SRC

** Environment variables

*** General

| Variable             | Default           | Description                            |
|----------------------+-------------------+----------------------------------------|
| =DATUM_BACKENDS=     | five (no oracle)  | Comma-separated list of backends       |
| =DATUM_STRICT=       | =0=               | =1= makes skips fatal (used in Docker) |
| =DATUM_USER=         | =root=            | Fallback DB user                       |
| =DATUM_PASS=         | (empty)           | Fallback DB password                   |
| =DATUM_SERVER=       | =localhost=       | Fallback DB server                     |
| =DATUM_DB=           | =datum_test=      | Fallback database name                 |
| =DATUM_PORT=         | (driver default)  | Fallback port                          |

*** MySQL-specific

| Variable             | Default           | Description                         |
|----------------------+-------------------+-------------------------------------|
| =DATUM_MYSQL_SERVER= | =DATUM_SERVER=    | MySQL server                        |
| =DATUM_MYSQL_USER=   | =DATUM_USER=      | MySQL user                          |
| =DATUM_MYSQL_PASS=   | =DATUM_PASS=      | MySQL password                      |
| =DATUM_MYSQL_DB=     | =DATUM_DB=        | MySQL database name                 |

*** PostgreSQL-specific

| Variable             | Default           | Description                         |
|----------------------+-------------------+-------------------------------------|
| =DATUM_PG_SERVER=    | =DATUM_SERVER=    | PostgreSQL server                   |
| =DATUM_PG_PORT=      | =5432=            | PostgreSQL port                     |
| =DATUM_PG_USER=      | =(whoami)=        | PostgreSQL user                     |
| =DATUM_PG_PASS=      | =DATUM_PASS=      | PostgreSQL password                 |
| =DATUM_PG_DB=        | =DATUM_DB=        | PostgreSQL database name            |

*** MSSQL-specific

| Variable             | Default           | Description                         |
|----------------------+-------------------+-------------------------------------|
| =DATUM_MSSQL_SERVER= | =DATUM_SERVER=    | MSSQL server                        |
| =DATUM_MSSQL_PORT=   | =1433=            | MSSQL port                          |
| =DATUM_MSSQL_USER=   | =sa=              | MSSQL user                          |
| =DATUM_MSSQL_PASS=   | =DATUM_PASS=      | MSSQL password                      |
| =DATUM_MSSQL_DB=     | =DATUM_DB=        | MSSQL database name                 |

*** Oracle-specific

| Variable               | Default          | Description                      |
|------------------------+------------------+----------------------------------|
| =DATUM_ORACLE_SERVER=  | =DATUM_SERVER=   | Oracle server                    |
| =DATUM_ORACLE_PORT=    | =1521=           | Oracle port                      |
| =DATUM_ORACLE_USER=    | =datum_test=     | Oracle user                      |
| =DATUM_ORACLE_PASS=    | =DATUM_PASS=     | Oracle password                  |
| =DATUM_ORACLE_SERVICE= | =FREEPDB1=       | Oracle service name (PDB)        |

Oracle is only tested when included in =DATUM_BACKENDS= (not in the
default set).  Use =COMPOSE_PROFILES=oracle= to start the Oracle
container.

*** SQLite-specific

| Variable             | Default               | Description                     |
|----------------------+-----------------------+---------------------------------|
| =DATUM_SQLITE_DB=    | =/tmp/datum_test.db=  | Path to SQLite database file    |

The =ansi= backend reuses PostgreSQL's connection variables (=DATUM_PG_*=).

* Test databases

Each backend has its own test SQL file with the same business domain
(customers, orders, products) but dialect-appropriate DDL:

| File               | Backend    | Tables | Views | Functions | Procedures |
|--------------------+------------+--------+-------+-----------+------------|
| =test-mysql.sql=   | MariaDB    |      8 |     3 |         3 |          4 |
| =test-postgres.sql=| PostgreSQL |      8 |     3 |         6 |          4 |
| =test-mssql.sql=   | SQL Server |      8 |     3 |         3 |          4 |
| =test-oracle.sql=  | Oracle     |      8 |     3 |         3 |          4 |
| =test-sqlite.sh=   | SQLite     |      4 |     1 |         — |          — |

SQLite's schema is intentionally simpler (no stored routines, no
multi-schema support).

The =ansi= backend has no dedicated test data file — it connects to
PostgreSQL's =datum_test= database using the generic INFORMATION_SCHEMA
driver, verifying that basic introspection works without dialect-specific
SQL.

* Introspection commands to verify

After connecting to any test database, these commands should work:

#+BEGIN_SRC
:databases
:schemas
:tables
:routines
:columns customers
:columns orders
:running
:user
:version
:pwd
:definition customers
:definition customer_order_summary
:definition format_currency
:definition datum_test
:use datum_test
:refresh
#+END_SRC
